7. FIX VOTING SYSTEM — Currently broken
The vote phase must work end to end. Here is the complete correct implementation:


Server side:

When phase changes to "vote", server initializes room.votes = {}
On vote:cast { targetId } from a player: record room.votes[playerId] = targetId, broadcast vote:update { votes: tallyVotes(room) } to all players in room so live counts update
Player can only vote once — if they emit vote:cast again ignore it
When all active players have voted OR 30s timer expires: server calls resolveVotes(roomId) which finds the player with most votes, emits vote:result { eliminatedId, wasImposter, imposterName } to room



Client side (VoteScreen.jsx):

Full screen overlay position: fixed, z-index: 1000 during vote phase
Background rgba(10,10,10,0.88)
Header: "VOTE FOR THE IMPOSTER" in grunge font
30 second countdown displayed large in red
All active non-eliminated players shown as large clickable blobs with their names
Clicking a blob: highlights it with red glow border, disables all other blobs, emits vote:cast { targetId }, shows "✓ Voted" label on selected blob
Live vote count badge on each blob updates as vote:update events arrive
Skip option: "Skip Vote" button counts as abstain
On vote:result received: animate reveal — blobs shake, voted-out player flips, show result text:

If innocent: "[Name] was NOT the imposter 😬" in yellow
If imposter caught: "[Name] WAS the imposter! Designers win! 🎉" in green
If 4 rounds done + imposter alive: "[Name] survived. Imposter wins! 😈" in red


Results show for 4 seconds then auto-advance to next phase
